JavaScript Fundamentals

What is JavaScript?

Javascript is a High-Level, Prototype-Based Object-Oriented, Multi-Paradigm, Interpreted Or Just-In-Time Compiled,Dynamic, Single-Threaded, Garbage-Collected Programming Language with First-Class Functions and a Non-Blocking Event Loop Concurrency Model.

Confused? Well I was at first, but it turns out you can boil it down into these simple statements:

  • JavaScript is a high-level programming language, which means it is designed to be easy for humans to read and write as you don't need to manage the memory.

  • JavaScript is prototype-based object-oriented, which means objects can inherit behaviors from other objects.

  • JavaScript is multi-paradigm, meaning there is more than one way to write it.

  • JavaScript is interpreted or just-in-time compiled, which means that it is compiled as it is needed, making it very fast.

  • JavaScript is dynamically typed, meaning that variables can be declared without a specific data type.

  • JavaScript is garbage-collected, which means that it automatically frees up memory that is no longer being used.

  • JavaScript is single-threaded, which means that only one piece of code can be executed at a time.

What is control flow and loops in JavaScript?

In computer science, control flow is the order in which statements are executed or performed in a program.

A loop is a repeating section of code that is executed a certain number of times until a condition is met.

To put this in perspective, control flow is like making a cake. You first start by preheating the oven, mixing the ingredients, pouring the batter into a pan, and baking for a set amount of time. The stages of the recipe are performed in a certain order.

A loop would be the stages of the recipe that need repetition. For example, when creaming the butter and sugar together, and because the ingredients start separated, you need to repeat the mixing process until the ingredients become one.

Accessing data in arrays and objects

Whether you are trying to pull data out or put it in, there is a difference when it comes to arrays and objects. Arrays are stored in a linear format, and accessed by indexed numbers, whereas objects are indexed key-value pairs, which allows them to be associated with a name.

To access the data in an object, you follow this syntax:

objectName.keyName, or in some cases objectName[keyName]

To access the data in an array, you follow this syntax:

arrayName[indexNumber]

JavaScript Functions

Functions are the main 'building blocks' of a program. They let programs call blocks of code, multiple times without repetition.

In JavaScript, there are 3 ways of writing a function.

Function declaration: A function that can be used before it's declared.

function exampleFunction(name){

return 'Hello' + name

}

Function expression: A function value stored in a variable.

const exampleFunction = function(name){

return 'Hello' + name

}

Arrow function: Good for one lines, but lacks some features provided by other types.

const exampleFunction = name => 'Hello' + name;
