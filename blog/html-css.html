Classes vs IDs

What are they?

Classes and IDs are two of the most important concepts in CSS, they are classified as selectors.

Selectors are what determine which elements in the DOM will be styled by the rules declaration properties. Selectors can match the ID, Class, Attribute or HTML element.

What is an ID Selector?

An ID is an unique identifier, used to select a single element on a page.

Usage

An ID selector finds the HTML element that matches its value based on the id attribute. It is best suited for uses where the element only appears on the page once (e.g. navigation-bar, header, footer).

Syntax

#idExample{

background-color: #fff;

}

What is a Class Selector?

A class selector is a way of grouping multiple elements together.

Usage

A Class selector finds the HTML element that matches its value based on the class attribute. It can be used for styling single or multiple elements that appear on the page multiple times (Headings, paragraphs, links).

Syntax

.classExample{

background-color: #fff;

}

What is specificity and why does it matter?

Specificity in CSS is the hierarchical structure of selectors. It is a formula which determines what style rule will be applied to an element.

When it comes to the specificity of Classes vs IDs, Classes have a lower specificity than IDs, meaning any conflicting styles applied to an element with a Class will be overridden by the styles applied with an ID.

Problem with IDs

ID selectors add a high level of unneeded specificity to your rule declarations while being non-reusable. Due to this, some organisations such as Airbnb have placed a blanket-ban on using ID selectors in their style sheets.
