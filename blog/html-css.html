Classes vs IDs

Q & A

What are they?

IDs and Classes are the two most important concepts of css. They are added to HTML elements in order to select and style them.

Why use an ID?

An ID is a unique identifier for an element and can only be used one time per page. They are best suited for when the element only needs to be selected once.

Once you apply an ID to your HTML via id="", you select it in your CSS with the # symbol followed by its given name.

Here's an example of how you select an ID.

#idExample{

background-color: #000

}

Why use a class?

A class is a way of grouping elements together, it used when you need the ability to style multiple elements.

A class is applied to HTML with class="", it is then selected in the CSS with the . symbol.

Here's an example of how you select a class.

.classExample{

background-color: #fff

}

What is specificity and why does it matter?

Specificity in CSS is the hierarchical structure of selectors. It is a formula which determines which style rules will be applied to an element.

When it comes to the specificity of Classes vs IDs, Classes have a lower specificity than IDs, meaning any conflicting styles applied to an element with a Class will be overridden by the styles applied with an ID.
