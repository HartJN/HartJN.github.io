Elegantly Solving A Problem

A while ago, I was working on a project where I had to analyse a large amount of data. I needed to be able to manipulate and generate reports with datasets stored in Excel spreadsheets. If you have worked with excel before, you know that it can be a very tedious process and not very optimised.

I knew that I could make this a lot easier using Python. I created a script that took the data from the spreadsheets and converted them into a Python dictionary and then created functions to analyse it.
By doing this, the whole process of analysing the data was much more efficient and faster

My Thoughts On Problem Solving Techniques

  • Pseudocode

    This is a great way to outline the steps you need to take in order to solve a problem. Being ables to write your code in plain english gives you a good idea of how to structure your code and helps you spot any potential errors before they occur.

  • Rubber ducky debugging

    There are a few reasons why this is a great problem solving technique. Being able to explain your code to another person(or duck in this case) really helps you understand what your code actually does and how each part works. Reading your code to another person, line by line, helps you spot errors that you may not see just by reading it in your head.

  • Console Debugging

    There are various methods you can use to debug within the console of a browser. You can use console.log() to print out values and track the flow of data, console.table() to print out tabular data, and console.trace() to print out a stack trace.

  • Googling

    This is your best friend when it comes to problem solving. The majority of issues you run into on a project, will have already been answered by another person, on a site like

  • Asking others

    this blends in well with the rubber ducky debugging method, but you get actual feedback as well. Sometimes it's easy to get stuck on one issue, and all that it needed to get past it is an outside point of view

  • Reflection

    Being able to reflect on what you have done at the end, is a good way to make sure that you know what you are talking about. You can use your reflections to help with spaced repetition.

  • Trying something

    This, in my opinion is the best way to try and solve a problem. By doing this, you go through a process of trial and error and get a grasp on what works and what doesn't.
